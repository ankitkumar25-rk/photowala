# PHOTOWALA - PROJECT DOCUMENTATION

## Project Overview
Photowala is a premium, full-stack e-commerce platform specializing in custom printing services (BillBooks, Digital Printing, Letterheads) and industrial machine services (CO2 Laser, Laser Marking, CNC Routing). It provides a seamless luxury experience for both standard product shopping and highly customizable service orders.

---

## Tech Stack

### Frontend (Storefront & Admin Panel)
- **Framework**: React 18 (Vite)
- **Styling**: Tailwind CSS & Vanilla CSS (Custom Luxury Theme)
- **State Management**: Zustand (Lightweight, persistent state)
- **Data Fetching**: TanStack Query (v5)
- **Routing**: React Router DOM (v6)
- **Iconography**: Lucide React
- **Animations**: Framer Motion & CSS Keyframes
- **HTTP Client**: Axios (with interceptors for CSRF & Token Auto-Refresh)

### Backend (Server)
- **Runtime**: Node.js
- **Framework**: Express.js
- **ORM**: Prisma
- **Database**: PostgreSQL
- **Security**: 
  - PASETO (Platform-Agnostic Security Tokens)
  - CSRF Protection (Custom middleware with double-submit cookies)
  - Bcrypt (Password hashing)
  - Helmet (Security headers)
- **Authentication**: 
  - JWT-replacement (PASETO) via httpOnly Cookies
  - Passport.js (Google OAuth 2.0 Integration)
- **Session/Cache**: Redis (via ioredis)

### Integrations & Services
- **Payments**: Razorpay (API & Webhooks)
- **Storage**: Cloudinary (Image management & CDN)
- **Email**: Nodemailer (SMTP integration for notifications)
- **Validation**: Zod (Schema validation for API and Forms)

---

## Architecture & Features

### 1. Dual-Interface System
- **Storefront**: A luxury-themed customer portal with product catalogs, wishlist, and service ledger.
- **Admin Panel**: A comprehensive dashboard for managing products, tracking industrial orders, and handling customer support.

### 2. Service Ledger System
- Custom-built workflow for tracking "Machine" and "Printing" orders.
- Dynamic form submission with file upload capabilities (Online/Email options).
- Real-time status tracking (Pending, Confirmed, Processing, Shipped, Delivered).

### 3. Security Model
- **Token Rotation**: 15-minute access tokens with 7-day refresh tokens.
- **Cookie Security**: httpOnly, Secure, and sameSite: 'Lax'/'None' configurations.
- **Role-Based Access Control (RBAC)**: Strict separation between `CUSTOMER`, `ADMIN`, and `SUPER_ADMIN`.

### 4. Database Schema
- **Users**: Central identity management with profile data.
- **Products & Categories**: Hierarchical structure with SKU and stock tracking.
- **Orders & ServiceOrders**: Unified tracking for standard sales and custom requests.
- **Payments**: Detailed transaction logs linked to Razorpay.

## Project Structure
- `/backend`: Express server, Prisma schema, and API controllers.
- `/frontend-store`: Customer-facing React application.
- `/frontend-admin`: Administrative dashboard React application.
- `/uploads`: Local temporary storage for file processing.
